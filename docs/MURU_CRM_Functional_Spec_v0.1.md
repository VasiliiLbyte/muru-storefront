# Функциональное описание CRM-админки MURU
## Подробное описание модулей, экранов и функций

**Версия:** 0.1 (черновик для предварительного показа заказчику)
**Дата:** 07.06.2026
**Приложение к:** ТЗ «Headless-платформа MURU» v0.1

---

## 1. О системе

CRM-админка MURU — это единая веб-панель управления магазином. Из одного места ведётся каталог, заказы, склад, клиенты, скидки и контент, и эти данные **одновременно питают и сайт, и Telegram Mini App**. Никаких отдельных таблиц и папок с фото: всё хранится в одной базе и редактируется через понятный интерфейс.

**Кто пользуется:** владелец магазина, менеджеры (обработка заказов, каталог, контент), кладовщик (склад). Доступ разграничен по ролям.

**Где работает:** браузер на компьютере (десктоп-ориентированный интерфейс), с адаптацией под планшет/телефон для базовых операций (например, посмотреть заказ со склада).

**Чем не является:** это не бухгалтерия. Система собирает все данные о продажах и формирует выгрузки, а учёт и отчётность ведёт бухгалтер заказчика в своём инструменте.

## 4. Дашборд (главная)

Сводка для быстрого старта рабочего дня:
- **Новые заказы** (требуют обработки) — счётчик и быстрый переход.
- **Выручка** за сегодня / неделю / месяц.
- **Заказы по статусам** (сколько ожидают оплаты, сборки, отправки).
- **Заканчивающиеся товары** — список SKU с низким остатком (порог настраивается).
- **Последние заказы** — мини-лента с переходом в карточку.
- Быстрые действия: создать заказ, добавить товар, оформить приёмку.

## 5. Каталог

### 5.1. Товары — список
Таблица всех товаров со столбцами: фото, артикул (SKU), название, категория, цена, скидка, остаток, статус (активен / скрыт / нет в наличии).

- **Поиск** по названию и артикулу.
- **Фильтры:** категория, статус, наличие (в наличии / нет / заканчивается), материал, тип, коллекция, диапазон цены, наличие скидки.
- **Сортировка** по любому столбцу (цена, остаток, дата обновления, название).
- **Постраничный вывод** и выбор количества строк.
- **Массовые действия** (выделить несколько/все): скрыть/показать, назначить категорию, поставить/снять скидку %, изменить цену (на %, на сумму), добавить в коллекцию, удалить.

### 5.2. Товары — карточка (создание / редактирование)
Поля сгруппированы по блокам:

**Основное**
- Название
- Артикул (SKU, например MU0001) — уникальный
- Ссылка (slug) — для URL, автоматически из названия, с ручной правкой
- Статус: активен / скрыт
- Единица измерения (по умолчанию «шт»)

**Категории**
- Главная категория и подкатегория — задают канонический адрес товара (важно для SEO, чтобы не было дублей)
- Дополнительные разделы — мультивыбор (товар показывается и там; например, подсвечник — и в «Вазы и аксессуары → Подсвечники», и в «Кухня и столовая → Подсвечники»)

**Цена и скидка**
- Цена, ₽
- Скидка, % (итоговая цена считается автоматически)
- Режим НДС *(на уточнение — вероятно «Без НДС» при УСН)*

**Остаток** (только просмотр, управляется в разделе «Склад»)
- Текущий остаток · Зарезервировано · Доступно к продаже

**Фотографии**
- Галерея фото с загрузкой (drag-and-drop), изменением порядка перетаскиванием, выбором главного фото
- Хранение в облаке (S3), быстрая отдача через CDN

**Описание**
- Текстовый редактор (абзацы, списки, выделение)

**Характеристики** (для фильтров на сайте и выгрузки в 1С)
- Цвет, Размер/габариты, Материал, Тип, Бренд, Дизайн, Страна-производитель, Происхождение, Упаковка, Коллекция
- Для текстиля: плотность ткани; для постельного: размеры наволочки / пододеяльника / простыни; для свечей/ароматов: ингредиенты

**Параметры доставки (СДЭК)**
- Вес, г · Габариты Д×Ш×В, см
- Источник значений: авто (рассчитано по материалу/размеру) или вручную — с защитой ручных значений от перезаписи

**SEO**
- Title, Description, H1
- Автогенерация по шаблону с возможностью переопределить вручную

### 5.3. Товары — импорт / экспорт
- **Импорт** из Excel/CSV (разовый перенос текущей таблицы и пакетные обновления) с предпросмотром и проверкой ошибок (дубли артикулов, пустые поля).
- **Экспорт** в Excel/CSV.
- **Экспорт в CommerceML** (формат обмена 1С) — задел под будущую интеграцию.

### 5.4. Категории
- **Дерево категорий** (два уровня) с перетаскиванием для изменения порядка и вложенности.
- Карточка категории: название, ссылка (slug, уникальная в пределах родителя), родитель, обложка, описание раздела, SEO-поля (Title/Description/H1), порядок, видимость.
- Показ количества товаров в каждой категории.

### 5.5. Характеристики (атрибуты)
- Справочник свойств (Тип, Материал, Цвет и т.д.) и их значений — чтобы фильтры на сайте были аккуратными, а не из разнобоя текста.
- *Глубина этого раздела зависит от набора нужных фильтров (на уточнение).*

## 6. Склад

Склад считает **штуки**. Источник правды по остаткам — журнал движений; остаток товара всегда равен сумме его движений. Это даёт прозрачность и историю.

### 6.1. Остатки
- Таблица товаров: текущий остаток, зарезервировано (под открытые заказы), доступно.
- Подсветка заканчивающихся и закончившихся позиций.
- Фильтр по категории/материалу, поиск, сортировка по остатку.

### 6.2. Приёмка (приход от поставщика)
- Документ приёмки: добавить товары и количества, указать поставщика (необязательно) и комментарий.
- Закупочная цена — необязательное поле «для справки» (без учёта себестоимости).
- При проведении остатки увеличиваются, в журнал пишется движение «приёмка».

### 6.3. Списание / корректировка
- Ручное уменьшение/изменение остатка с обязательным указанием причины (брак, потеря, исправление).
- Пишет движение «корректировка».

### 6.4. Инвентаризация
- Ввод фактически пересчитанных количеств по списку товаров.
- Система показывает расхождения (учётное vs фактическое) и при подтверждении выравнивает остатки, фиксируя движения.

### 6.5. Журнал движений
- Полная неизменяемая история всех операций: приёмки, списания, отгрузки по заказам, резервы, корректировки.
- Фильтры по товару, типу операции, дате, исполнителю. Это и аудит, и основа для будущей выгрузки в 1С.

### 6.6. Резервы
- Просмотр, что и под какие заказы зарезервировано.
- Логика: при оформлении заказа товар резервируется, при оплате/сборке — списывается, при отмене — резерв снимается (правила переходов настраиваются вместе со статусами заказа).

## 7. Заказы

### 7.1. Список
Таблица заказов: номер, дата, клиент, сумма, статус заказа, статус оплаты, источник (сайт / Mini App), способ доставки.

- **Фильтры:** статус, статус оплаты, источник, способ доставки, диапазон дат, наличие промокода.
- **Поиск** по номеру, имени/телефону клиента.
- Сортировка, постраничный вывод.

### 7.2. Карточка заказа
- **Состав заказа:** позиции (товар, кол-во, цена, цвет/размер); возможность добавить/убрать позицию и изменить количество — сумма пересчитывается **на сервере** (клиентским суммам доверия нет).
- **Клиент:** имя, телефон, ссылка на карточку клиента.
- **Доставка:** способ (СДЭК — курьер/ПВЗ), адрес или пункт выдачи, тариф, трек-номер (подтягивается из СДЭК), плановый срок.
- **Оплата:** статус из ЮКассы, оплаченная сумма, дата, ссылка на платёж/чек; устойчивость к пропущенным вебхукам (статус допроверяется напрямую).
- **Скидки:** применённый промокод и сумма скидки.
- **Внутренний комментарий** менеджера, назначение ответственного менеджера.
- **История заказа:** хронология смен статусов и изменений.
- Действия: сменить статус, оформить возврат, распечатать/выгрузить заказ.

### 7.3. Статусы заказа и переходы
Предлагаемый рабочий процесс (настраивается):

| Статус | Что происходит |
|---|---|
| Черновик | Корзина не оформлена |
| Новый (оформлен) | Заказ создан → товары резервируются, уведомление менеджеру |
| Оплачен | Оплата подтверждена ЮКассой → списание со склада |
| Собран | Готов к передаче в доставку |
| Отправлен | Передан в СДЭК, есть трек-номер → уведомление клиенту |
| Доставлен | Завершён |
| Отменён | Резерв/списание возвращается на склад |
| Возврат | Оформлен возврат (полный/частичный) |

Смена статуса автоматически влияет на склад и уведомления.

### 7.4. Создание заказа вручную
- Для заказов по телефону/в переписке: выбрать клиента (или создать), добавить товары, указать доставку, создать заказ и при необходимости выставить оплату.

### 7.5. Возвраты
- Оформление возврата по заказу (полный или частичный): выбор позиций, причина, возврат товара на склад, инициирование возврата средств через ЮКассу.

### 7.6. Уведомления по заказам
- Менеджеру — о новом заказе (Telegram + email, как сейчас).
- Клиенту — о смене ключевых статусов (оплачен, отправлен с трек-номером).
- Шаблоны и каналы настраиваются в «Настройках».

## 8. Клиенты

### 8.1. Список
- Таблица: имя, телефон/e-mail, источник (сайт / Mini App), число заказов, сумма покупок, дата последнего заказа.
- Поиск и фильтры (по сумме, по дате последнего заказа, по сегменту).

### 8.2. Карточка клиента
- Контактные данные, привязанные аккаунты (Telegram и/или веб — объединяются по телефону/e-mail).
- История заказов, сохранённые адреса доставки.
- Сегменты/теги, заметки менеджера.

### 8.3. Сегменты
- Группировка клиентов (например, «постоянные», «оптовые») для адресных скидок и акций.

## 9. Скидки и акции

### 9.1. Промокоды (реализовано в Mini App, переносится в CRM)
- Список промокодов и создание: код, тип (процент / фиксированная сумма), значение, минимальная сумма заказа, период действия (с/по), лимит использований (общий и на пользователя), активность.
- Статистика использования по каждому коду.

### 9.2. Скидки на товары
- Скидка % на конкретный товар (также доступна в карточке товара).
- Массовое назначение скидки выделенным товарам или категории.

### 9.3. Акции (правила)
- Правила: скидка на категорию / выборку товаров / сегмент клиентов, с расписанием (старт и финал).
- Приоритеты при пересечении акций.

### 9.4. Раздел «Распродажа»
- Автонаполнение витринного раздела «Распродажа» товарами со скидкой — без ручного ведения отдельного списка.

## 10. Контент (CMS)

Контент управляет и сайтом, и Mini App из одного места. Все элементы имеют поля видимости и SEO (где применимо).

### 10.1. Баннеры
- Баннеры главной: изображение, заголовок/подзаголовок, ссылка, порядок, расписание показа, активность.

### 10.2. Коллекции (раздел «Коллекции» / лендинги)
- Создание коллекции: название, обложка, описание, привязка товаров, SEO-поля.
- Из текущей таблицы — коллекции «Тёплый песок», «Лето в доме».

### 10.3. Вдохновение (лукбуки)
- Визуальные подборки-вдохновение (раздел «Вдохновение» на сайте): изображения, описание, ссылки на товары.

### 10.4. Страницы
- Статичные страницы с текстовым редактором и SEO: О нас, Контакты, Реквизиты, Клиентам (помощь/доставка/оплата), Гид по подаркам, Оферта, Политика конфиденциальности.

### 10.5. Блог *(опционально, для контентного SEO)*
- Статьи: заголовок, текст, обложка, рубрика, дата, автор, SEO-поля. Полезно для продвижения по информационным запросам.

### 10.6. Подарочные карты *(на уточнение)*
- На сайте есть раздел «Подарочные карты». Нужно уточнить механику (номинал, генерация кода, погашение при заказе) — это отдельный мини-функционал, объём зависит от требований.

## 11. Выгрузки и отчёты

### 11.1. Выгрузка для бухгалтера
- Выгрузка продаж/возвратов за период (Excel/CSV) со всеми полями, нужными для учёта: дата, номер заказа, позиции, суммы, способ оплаты, статус, ссылка на фискальный чек ЮКассы.

### 11.2. Экспорт CommerceML (готовность к 1С)
- Выгрузка каталога, остатков, цен и заказов в формате CommerceML — чтобы при появлении 1С подключение шло штатным обменом, а не переделкой.

### 11.3. Отчёты
- Продажи за период, топ товаров, отчёт по остаткам (в штуках), список заканчивающихся товаров.

## 12. Настройки

- **Реквизиты и контакты** магазина (используются на сайте и в документах).
- **Доставка:** параметры СДЭК — адрес отправителя (Санкт-Петербург, ул. Дубровская, д.13), тарифы, габариты/вес по умолчанию.
- **Оплата:** параметры ЮКассы, состав фискального чека (54-ФЗ), режим НДС.
- **Пользователи и роли:** создание сотрудников, назначение ролей.
- **SEO-шаблоны:** шаблоны мета-тегов для товаров и категорий.
- **Юридические документы:** оферта, политика, версия согласия на обработку ПДн (152-ФЗ).
- **Уведомления:** каналы (Telegram, email), получатели, шаблоны; приветственное сообщение бота.

---

## 13. Сквозные элементы

- **Глобальный поиск** по заказам, товарам, клиентам.
- **Журнал действий (аудит):** кто и что менял в ключевых разделах.
- **Адаптивность:** базовые операции доступны с планшета/телефона.
- **Производительность:** быстрые списки с фильтрами, отложенная загрузка изображений.
- **Безопасность:** разграничение прав, серверный пересчёт цен и сумм, данные на серверах в РФ.

---

## 14. Что требует уточнения у заказчика
1. **Подарочные карты** — нужна ли механика и какая (номинал, коды, погашение).
2. **Авторизация на сайте** — по SMS (нужен платный СМС-провайдер) или e-mail/пароль.
3. **Режим НДС / система налогообложения** (вероятно УСН) — влияет на чек ЮКассы.
4. **Глубина характеристик/фильтров** — какой набор фильтров нужен на сайте.
5. **Точная матрица прав** по ролям (раздел 3).
6. **Вариативность товара** — оставляем «1 артикул = 1 товар» (рекомендуется) или делаем полноценные вариации.
